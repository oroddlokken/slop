# Write Agent Docs

Guidelines for writing and updating agent-facing documentation (CLAUDE.md, AGENTS.md, copilot-instructions.md, .claude/rules/). Load this skill when writing, editing, or updating agent instructions.

## What you get

A loaded set of principles that guide how agent docs are written:

- **Positive framing** — directives outperform prohibitions (the "Pink Elephant" effect)
- **Include reasoning** — rules with a "why" generalize to novel situations
- **No weak language** — eliminates "should", "consider", "try to" escape hatches
- **Token budget** — keeps files lean (<200 lines) so compliance stays high
- **Information architecture** — puts content in the right place (CLAUDE.md vs .claude/rules/ vs skills)
- **No self-evident rules** — agents can read code; only document what they can't derive

## When to use

- Writing a new CLAUDE.md, AGENTS.md, or copilot-instructions.md
- Updating existing agent instructions
- Adding .claude/rules/ files
- Reviewing agent docs for quality

Tell your agent to read this repository and ask it to help you integrate it into your Claude Code setup as a skill.
